## Новые возможности

### 1. Task Queue (Очередь задач)

Задачи теперь ставятся в очередь и выполняются последовательно. Пока одна задача выполняется (`running`), остальные ожидают в статусе `queued`. После завершения текущей задачи автоматически запускается следующая.

**Статусы задач:**
| Статус | Описание |
|---|---|
| `queued` | В очереди, ожидает выполнения |
| `running` | Выполняется агентом |
| `waiting_confirm` | Агент ожидает подтверждения пользователя |
| `completed` | Успешно завершена |
| `error` | Завершена с ошибкой |
| `cancelled` | Отменена пользователем |

**API:**
- `GET /api/queue` — список задач в очереди/истории текущей сессии
- `POST /api/queue/cancel` — отмена задачи `{ taskId }`

### 2. Session-Aware архитектура

Каждая задача привязана к `sessionId`. Сессии изолируют:
- Preview (скриншот браузера)
- DOM Snapshot
- Selected Element
- Телеметрию и историю действий
- Step Snapshots для replay

**API:**
- `GET /api/sessions` — список всех сессий
- `GET /api/sessions/:id` — детали сессии

**UI:**
- Панель «Сессии» в левой боковой панели
- Кнопка «+» для создания новой сессии
- Клик по сессии переключает контекст
- Счётчик задач рядом с каждой сессией

### 3. Replay (Воспроизведение шагов)

Каждый шаг агента сохраняется в таблицу `step_snapshots` вместе со скриншотом и DOM snapshot. Это позволяет просматривать пошаговое выполнение задачи.

**Возможности:**
- Просмотр скриншота на каждом шаге
- Список шагов с действием, статусом, временной меткой
- Навигация: Первый / Предыдущий / Play / Следующий / Последний
- Прогресс-бар
- Автовоспроизведение с интервалом

**API:**
- `GET /api/tasks/:id/steps` — список шагов задачи
- `GET /api/tasks/:id/steps/:stepIndex/screenshot` — скриншот конкретного шага

**UI:**
- Вкладка REPLAY в правой панели
- Клик по завершённой задаче в очереди открывает replay

### 4. Risk Card (Карточка подтверждения)

При работе в режиме безопасности «Подтверждение» (`confirm`), агент останавливается перед потенциально опасными действиями и показывает карточку подтверждения.

**Уровни риска:**
| Уровень | Цвет | Пример |
|---|---|---|
| Низкий (`low`) | Зелёный | Навигация, чтение |
| Средний (`medium`) | Оранжевый | Заполнение email/логина |
| Высокий (`high`) | Красный | Submit, Delete, оплата |

**Действия, требующие подтверждения:** `click_button`, `click_link`, `fill_input`, `select_option`, `submit_form`

**API:**
- `POST /api/agent/confirm` — ответ `{ taskId, approved: true/false }`
- `GET /api/agent/confirm/status` — проверка наличия ожидающего подтверждения

**UI:**
- Полноэкранный оверлей с размытием фона
- Иконка и цвет зависят от уровня риска
- Отображение действия, параметров и причины оценки
- Кнопки «Подтвердить» / «Отклонить»
- Автоматический таймаут 120 секунд → отклонение

### 5. Режимы безопасности

Три режима работы агента, переключаемые в UI:
- **Только чтение** (`readonly`) — агент только собирает информацию
- **Подтверждение** (`confirm`) — агент спрашивает перед действиями (показывает Risk Card)
- **Полный доступ** (`full`) — все действия выполняются автоматически

## Схема БД (новое)

```sql
-- Расширение agent_tasks
ALTER TABLE agent_tasks ADD COLUMN session_id TEXT DEFAULT 'default' NOT NULL;
ALTER TABLE agent_tasks ADD COLUMN queue_position INTEGER DEFAULT 0 NOT NULL;
-- Статус расширен: queued | running | waiting_confirm | completed | error | cancelled

-- Новая таблица
CREATE TABLE step_snapshots (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id INTEGER NOT NULL,
  session_id TEXT DEFAULT 'default' NOT NULL,
  step_index INTEGER NOT NULL,
  phase TEXT DEFAULT '' NOT NULL,
  action TEXT DEFAULT '' NOT NULL,
  status TEXT DEFAULT 'info' NOT NULL,
  detail TEXT DEFAULT '' NOT NULL,
  timestamp TEXT NOT NULL,
  screenshot_base64 TEXT,
  snapshot_json TEXT
);
```

## Ограничения и известные особенности

1. **Risk Card** показывается только в режиме «Подтверждение» (`confirm`). В режиме «Только чтение» агент не выполняет действий, требующих подтверждения.
2. **Replay скриншоты** сохраняются как Base64 в SQLite. При большом количестве задач база данных может существенно вырасти.
3. **Автовоспроизведение replay** — интервал 2 секунды между шагами, не настраивается из UI.
4. **Сессии** не имеют ограничения по количеству, но каждая занимает отдельный браузерный контекст в памяти агента.
5. **Таймаут подтверждения** — 120 секунд. Если пользователь не ответит, действие автоматически отклоняется.
6. **Миграции** выполняются автоматически при запуске сервера (добавление столбцов `session_id`, `queue_position`, создание таблицы `step_snapshots`).
