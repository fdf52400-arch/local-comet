# Local Comet — Итерация 4: Summary

## Версия: v0.4.0

## Новые возможности

### 1. Live Browser Preview
- Встроенная панель Preview в правой части UI с имитацией «браузерного окна» (хром с адресной строкой и кнопками)
- Отображение текущего URL агента в реальном времени
- Автообновление скриншота после каждого действия агента (navigate, dom_snapshot, click_link, и т.д.)
- Кнопка ручного обновления (Refresh) для принудительного снятия скриншота
- Индикатор PREV в верхней панели при наличии активного скриншота
- Sync bar с номером синхронизации, текущим действием и временной меткой

### 2. Preview + Snapshot Sync
- Единый `PreviewState` объект на сервере: screenshotBase64 + snapshot + url + syncId + currentAction
- Монотонный счётчик `syncId` для отслеживания версий preview
- SSE-событие `preview_update` отправляется после каждого действия агента
- Пользователь видит, что screenshot, URL и DOM snapshot относятся к одной версии страницы
- Sync bar показывает номер синхронизации и время последнего обновления

### 3. Выбор элементов из DOM Snapshot
- Клик по элементу в карте DOM выделяет его (primary-цвет, бордер, иконка Target)
- Баннер «Выбран: [тип] текст» появляется над списком элементов с кнопкой сброса
- Бэйдж "selected" на выбранном элементе
- Выбор элемента отправляется на backend через `POST /api/element/select`
- Для выбранного элемента доступны быстрые действия через Action Console:
  - Автозаполнение типа действия (click_link/click_button/fill_input) при выборе элемента
  - Автозаполнение текста/placeholder из выбранного элемента

### 4. Action Console
- Панель ручных действий на вкладке ACTION в правой панели
- Выбор типа действия из выпадающего списка (9 действий):
  - Навигация, клик по ссылке, нажать кнопку, заполнить поле
  - Извлечь текст, найти ссылки, найти кнопки, DOM Snapshot, суммаризировать
- Контекстные параметры в зависимости от типа действия (URL / текст / placeholder + value)
- Кнопка «Выполнить» с индикатором загрузки
- Результат выполнения отображается inline (SUCCESS/ERROR с деталями)
- Результат ручного действия попадает в общий журнал телеметрии как `manual_action`
- Preview автоматически обновляется после ручного действия
- Автозаполнение параметров из выбранного DOM-элемента

### 5. Усиленный Replanner
- Агент учитывает выбранный пользователем элемент как приоритет при планировании
- Если пользователь выбрал ссылку — агент сначала пробует click_link на неё
- Если пользователь выбрал кнопку — агент сначала пробует click_button
- Если пользователь выбрал поле — агент предлагает fill_input
- При неудаче действия — автоматический поиск альтернативного элемента (findAlternativeAction):
  - Для click_link: поиск похожей ссылки по тексту
  - Для click_button: первая доступная кнопка
  - Для fill_input: альтернативное поле ввода
- Новое действие `click_button` добавлено в набор разрешённых
- Умный поиск полей: если цель содержит «поиск/найти», ищет input с placeholder search/q/query
- Контекст выбранного элемента передаётся в модель через `reasonWithModel`

### 6. Новые Backend-маршруты

| Маршрут | Метод | Описание |
|---|---|---|
| `/api/preview` | GET | Текущее состояние preview (URL, syncId, snapshot, hasScreenshot) |
| `/api/preview/screenshot` | GET | JPEG-скриншот текущей страницы (image/jpeg) |
| `/api/preview/refresh` | POST | Принудительное обновление preview |
| `/api/action/execute` | POST | Выполнение ручного действия из UI |
| `/api/element/select` | POST | Установка выбранного элемента |
| `/api/element/selected` | GET | Получение текущего выбранного элемента |

## Ограничения

- Preview использует серверные JPEG-скриншоты (не live видеопоток) — обновляется после каждого шага
- Drag-and-drop element picker поверх реальной страницы не реализован (по условию)
- Нет мультивкладочного браузера
- Нет встраивания живого Chromium окна (по условию)
- Manual action для fill_input требует ввода placeholder и value вручную (или автозаполнение из выбранного элемента)
- Electron-перенос не делался (по условию)
